# Portfolio Optimization: Overfitting Demonstration and Advanced Methods

This repository implements a comprehensive suite of portfolio optimization experiments demonstrating overfitting in classical mean-variance optimization and advanced techniques to address it using the skfolio library.

## Overview

The repository contains four experiments that progressively demonstrate:
1. **Overfitting in Mean-Variance Optimization**: In-sample vs out-of-sample efficient frontier comparison
2. **Factor Model Prior Integration**: Using ridge regression to incorporate factor structure
3. **Stacking Optimization**: Ensemble methods with walk-forward cross-validation
4. **API Validation**: Scikit-learn compatibility and workflow integration

## Experiments

### Experiment 1: Overfitting Demonstration

Demonstrates that classical mean-variance optimization overfits by comparing in-sample and out-of-sample efficient frontiers.

**Key Parameters:**
- Dataset: S&P 500 daily returns
- Train/Test Split: 80/20 chronological
- Constraints: AAPL max weight 20%
- L2 Regularization: λ = 0.01
- Frontier Points: 100

**Expected Outcome:** Training efficient frontier dominates test frontier, demonstrating overfitting.

### Experiment 2: Factor Model Prior Integration

Integrates factor model priors using ridge regression into mean-variance optimization.

**Key Parameters:**
- Ridge Regression: α = 0.1
- Factor Dataset: Multi-factor returns aligned with equity data
- Comparison: Baseline vs Factor-Prior MeanRisk

**Expected Outcome:** Factor model provides more stable out-of-sample performance.

### Experiment 3: Stacking Optimization

Implements ensemble portfolio optimization using multiple base estimators.

**Key Parameters:**
- Base Estimators: InverseVolatility, MeanRisk(CVAR), HierarchicalRiskParity
- Final Estimator: MeanRisk(CVAR)
- Walk-Forward: train_size=252, test_size=60
- Benchmark: EqualWeighted

**Expected Outcome:** Stacking ensemble shows improved robustness vs equal-weighted benchmark.

### Experiment 4: API Validation

Validates scikit-learn-style API implementation across all estimators.

**Tests:**
- fit/predict/transform consistency
- Parameter getting/setting
- Cross-validation compatibility
- No data leakage
